Section 01

Document Structure Elements

These elements form the skeleton of every HTML page. You can't see most of them on screen — they work behind the scenes to tell the browser how to interpret your page.

Note: The structural elements <!DOCTYPE html>, <html>, <head>, <meta>, <title>, <link>, <style>, and <script> are all actively used in this very file. View the page source (Ctrl+U) to see them.

What Each Structural Element Does

Document Structure Elements
Element Location Purpose
<!DOCTYPE html>Line 1Declares HTML5 to the browser
<html lang="en">Wraps everythingRoot element + language declaration
<head>Before <body>Metadata container — not rendered visually
<title>Inside <head>Browser tab title
<meta charset>Inside <head>Character encoding (UTF-8 = all languages)
<meta viewport>Inside <head>Makes page mobile-responsive
<link>Inside <head>Links CSS stylesheets or fonts
<style>Inside <head>Internal CSS styles
<script>End of <body>JavaScript code or file
<base>Inside <head>Base URL for all relative links
<body>After <head>All visible page content lives here
 
<!-- The skeleton of EVERY HTML page -->
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title Here</title>
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <!-- All visible content goes here -->
  </body>
</html>

Headings — h1 through h6

HTML has six heading levels. Use them in order — never skip levels just for visual size. Headings define your page's outline for screen readers and search engines.

h1 — Main Page Title (only one per page)

h2 — Major Section Heading

h3 — Subsection Heading

h4 — Sub-subsection Heading

h5 — Minor Heading (rarely used)
h6 — Smallest Heading Level

    Semantic HTML Elements

    Semantic elements clearly describe their role — to the browser, developers, and screen readers. Always prefer these over generic <div> elements.

    Did you know? This entire page uses semantic HTML. The top bar is a <header>, the jump links are a <nav>, this content area is <main>, each topic block is a <section>, and the bottom bar is a <footer>.

    article

    My First HTML Project

    Published: March 1, 2024

    The <article> element represents a self-contained piece of content that could stand alone — like a blog post, news article, or product card.

    Read more →

    aside

    💡 Did You Know?

    The <aside> element is for content that is related to, but separate from, the main content — like sidebars, callout boxes, and related links.

    details & summary

    What the difference between <strong> and <b>?

    <strong> means the text is important — screen readers will stress it. <b> is visually bold with no semantic meaning. Always prefer <strong> for actual importance.

    What is the difference between <em> and <i>?

    <em> means the text is emphasized. <it; is visually italic only — often used for technical terms or icon fonts.

    This one starts open (has the open attribute)

    Add the open attribute to <details> to make it expanded by default.

    Generic Containers: <div> & <span>

    Rule: Always ask if a semantic element fits first. Only use <div> or <span> when no semantic element is appropriate.

    div — Block Container

    This is a <div> — it creates a block-level box. It starts on a new line and takes up the full width.

    A second <div>. Each div starts on its own line.

    span — Inline Container

    A <span> is inline — it stays inside the text flow without creating a new line. It's used to style a specific word or phrase within a paragraph.

    Inline <object> & <embed>

    <object> and <embed> are used to embed external resources like PDFs. Example syntax:

    <!-- Embed a PDF -->
<object data="document.pdf" type="application/pdf" width="600" height="400">
  <p>PDF cannot be displayed. <a href="document.pdf">Download instead</a>.</p>
</object>

<!-- Alternative with embed -->
<embed src="document.pdf" type="application/pdf" width="600" height="400">
